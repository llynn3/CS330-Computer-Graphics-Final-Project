How do I approach designing software?

I approach designing software by first understanding the overall goal of the project and then breaking it into smaller, manageable components. For this project, I identified the objects I wanted to recreate from my reference image and planned which basic 3D shapes would best represent each object before beginning development. This made the implementation process much more organized and easier to manage.

What new design skills has your work on the project helped you to craft?

This project strengthened my ability to think about how complex objects can be built from simple geometric shapes. It also improved my understanding of object placement, scaling, textures, lighting, and camera positioning to create a cohesive 3D scene.

What design process did you follow for your project work?

I followed an iterative design process by creating one object at a time, testing its appearance, adjusting its size and position, and then moving on to the next object. After the scene layout was complete, I refined the textures, lighting, and camera controls to improve the overall presentation.

How could tactics from your design approach be applied in future work?

Breaking a large project into smaller tasks is a strategy I can apply to future software development projects. Designing, testing, and refining each component individually makes complex projects more manageable and easier to troubleshoot.
